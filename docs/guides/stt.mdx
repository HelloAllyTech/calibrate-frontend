# Speech-to-Text Evaluation

Pense allows you to evaluate multiple Speech-to-Text (STT) providers simultaneously using your own audio samples. This helps you find the best provider for your specific language and use case.

## What You'll Learn

- Upload audio samples with reference transcriptions
- Compare multiple STT providers side-by-side
- Analyze accuracy metrics (WER, String Similarity, LLM Judge)
- Measure latency (TTFB, Processing Time)

## Step 3: Upload Your Dataset

### Dataset Tab

**Option A: Manual Upload**

1. Click **Upload audio file** to add a .wav file
2. Enter the **ground truth transcription**
3. Click **Add another sample** for more

**Option B: ZIP Upload**

Upload a ZIP file containing:
```
audios/
  sample1.wav
  sample2.wav
data.csv
```

Where `data.csv` has:
```csv
audio,transcription
audios/sample1.wav,"Hello, how are you today?"
audios/sample2.wav,"I'd like to schedule an appointment."
```

<Tip>
Click **Download sample ZIP** to get a template with the correct format.
</Tip>

## Understanding the Metrics

| Metric | Description | Preference |
|--------|-------------|------------|
| **WER** | Word Error Rate - measures transcription accuracy | Lower is better |
| **String Similarity** | Character-level similarity (0-1) | Higher is better |
| **LLM Judge Score** | Semantic accuracy evaluation | Pass is better |
| **TTFB** | Time to First Byte | Lower is better |
| **Processing Time** | Total transcription time | Lower is better |

<Info>
**WER (Word Error Rate)** is calculated as: (Insertions + Deletions + Substitutions) / Total Words. A WER of 0.05 means 5% of words were incorrectly transcribed.
</Info>

## Best Practices

<AccordionGroup>
  <Accordion title="Use representative audio samples">
    Include samples that match your real-world use case: similar accents, background noise levels, and speaking speeds.
  </Accordion>
  <Accordion title="Test edge cases">
    Include challenging samples: technical terms, names, numbers, and code-switching between languages.
  </Accordion>
  <Accordion title="Consider latency for real-time applications">
    For voice agents, TTFB is critical. A provider with slightly lower accuracy but much faster response may be better.
  </Accordion>
</AccordionGroup>
